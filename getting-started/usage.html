# Usage

Below is a basic example to check if your Vuelr installation is working:

<template>
  <Vuelr :code="code" v-slot="{ target }">
    <div :id="target" />
    <textarea v-model="code" />
  </Vuelr>
</template>

<script lang="ts">
export default defineComponent({
  setup() {
    return {
      code: ref('<p>Hello world!</p>')
    };
  }
});
</script>

Vuelr includes very basic <textarea> and preview elements for testing the component. Vuelr does not include any sophisticated editor or styling.

It's now up to you to add this using the slots Vuelr exposes. When you populate the default slot, the basic elements are disabled.

The example below uses CodeMirroropen in new window. You can also view the complete source code of this component hereopen in new window.

# Props

NameTypeDefaultDescription
codestringThe code for Vuelr to render.
value / v-modelstringThe code for Vuelr to render. Only applies if you use Vuelr without default slot content.

# Events

EventArgumentsDescription
update:modelValue
valueThe value the textarea was changed to.
Only applies if you use Vuelr without default slot content.

# Slots

SlotProperties
default
targetThe element ID Vuelr tries to render the dynamic component to.
iterationEvery change of the code increases iteration by one.
errorThe current Vuelr error.
compiledIncludes the compiled template, script and style as strings.

TIP

Below is an example on how you can access the slot properties.

<Vuelr :code="code" v-slot="{ target, error, iteration, compiled }">
  <ul>
    <li>{{ target }}</li>
    <li>{{ error }}</li>
    <li>{{ iteration }}</li>
    <li>{{ compiled }}</li>
  </ul>
</Vuelr>
